# Plan: Add `## [Unreleased]` Section to Changelog Convention

## Context

Many projects follow [Keep a Changelog](https://keepachangelog.com/) by placing a
`## [Unreleased]` section at the top of CHANGELOG.md with a clickable compare link
(`latest-tag...HEAD`). The link is defined as a **Markdown reference-style link** at
the bottom of the file, keeping the header clean: just `## [Unreleased]`.

The current CHANGELOG.md lacks this section, and also has a structural bug: the
preamble text is at the **bottom** under a duplicate `## Changelog` heading.

## Changes

### 1. `CHANGELOG.md` — Fix structure + add Unreleased section

Target structure:

```markdown
# Changelog

All notable changes to this project will be documented in this file.
Versions follow [Semantic Versioning](https://semver.org).

<!-- Entries below are maintained automatically by release-please and edited manually -->

## [Unreleased]

## [1.1.0](https://github.com/byheaven/byheaven-skills/compare/byheaven-skills-1.0.0...byheaven-skills-1.1.0) (2026-03-17)

…existing 1.1.0 content unchanged…

<!-- Link definitions -->
[unreleased]: https://github.com/byheaven/byheaven-skills/compare/byheaven-skills-1.1.0...HEAD
```

- Move preamble from bottom to top (under `# Changelog`)
- Delete the duplicate `## Changelog` heading and its content at the bottom
- Insert empty `## [Unreleased]` section before `## [1.1.0]`
- Add reference-style link definition at the very end of the file
- Version headers keep their existing inline links (generated by release-please)

### 2. `docs/changelog-style-guide.md` — Document Unreleased convention

Add a new sub-section **"Unreleased Header — Update Link Each Release"** after the
existing "Version Header — NEVER MODIFY" section (~line 113). Content:

- The header is just `## [Unreleased]` — clean, no inline URL
- The compare URL is a reference-style link definition at the bottom of the file
- When editing a Release PR: update the `[unreleased]:` link definition to compare
  from the **new** version's tag (e.g., `...compare/byheaven-skills-1.2.0...HEAD`)
- The tag doesn't exist yet during PR editing — it's created on merge. This is expected.
- release-please ignores this line entirely

Also add to the Final Checklist:

- `[ ] [unreleased] link definition at bottom of file updated to new version tag`

## How release-please Coexists

release-please inserts new `## [x.y.z]` sections before the first existing `## [x.y.z]`
heading. It skips `## [Unreleased]` because it doesn't match the versioned pattern.
Result: new versions land **between** `[Unreleased]` and the previous version — exactly
the correct Keep a Changelog behavior.
